# Glossary
## AI Job Agent — Automated Job Application System
**Version:** 1.0

---

| Term | Definition |
|------|-----------|
| **ATS** | Applicant Tracking System — software used by employers to manage job applications. Scans resumes for keywords before human review. |
| **ATS Score** | A simulated score (0–100) predicting whether a tailored resume will pass ATS screening for a specific job. |
| **Access Token** | Short-lived JWT (24h) used to authenticate API requests. |
| **AI Agent** | A specialised software module that uses LLM capabilities to perform a specific task within the pipeline (e.g., analyse, tailor, generate). |
| **AMQP** | Advanced Message Queuing Protocol — the messaging protocol used by RabbitMQ. |
| **Apache Tika** | A Java library for detecting and extracting content from PDF, DOCX, and other file formats. |
| **BCrypt** | A password hashing function used to store user passwords securely. |
| **BRD** | Business Requirements Document — defines business objectives, stakeholders, scope, and success criteria. |
| **Circuit Breaker** | A design pattern that detects failures (e.g., AI API down) and stops making calls for a cooldown period to prevent cascading failures. |
| **Cover Letter** | A personalised letter accompanying a job application, generated by the AI to complement the tailored resume. |
| **CRUD** | Create, Read, Update, Delete — the four basic database operations. |
| **CSP** | Content Security Policy — HTTP header that helps prevent XSS attacks by controlling which resources the browser can load. |
| **Daily Digest** | An email sent once per day summarising the top job matches for a user. |
| **Dead Letter Queue (DLQ)** | A RabbitMQ queue that holds messages that could not be processed after the maximum number of retry attempts. |
| **Docker Compose** | A tool for defining and running multi-container Docker applications using a YAML file. |
| **Domain Score** | Component of the match score (10% weight) measuring alignment between the job's industry domain and the user's experience domain. |
| **DTOs** | Data Transfer Objects — simple classes used to transfer data between layers (controller ↔ service ↔ client). |
| **Experience Score** | Component of the match score (25% weight) based on how closely the user's years of experience match the job's requirement range. |
| **FastAPI** | A modern, high-performance Python web framework for building APIs. Used for the scraper service. |
| **GDPR** | General Data Protection Regulation — EU regulation governing data privacy. Key requirements: right to erasure, data portability, breach notification. |
| **GPT-4o** | OpenAI's flagship large language model used for high-quality resume tailoring and cover letter generation. |
| **GPT-4o-mini** | OpenAI's smaller, faster, cheaper model used for JD analysis where top quality is less critical. |
| **HLD** | High-Level Design — architectural overview of the system including components, interactions, and technology stack. |
| **Hikari** | A high-performance JDBC connection pool library used by Spring Boot. |
| **HSTS** | HTTP Strict Transport Security — header instructing browsers to only access the site over HTTPS. |
| **InnoDB** | MySQL's default storage engine; supports ACID transactions and foreign keys. |
| **JD** | Job Description — the text description of a job posting, including responsibilities, requirements, and qualifications. |
| **JPA** | Java Persistence API — Java specification for ORM, implemented by Hibernate in this project. |
| **JSON** | JavaScript Object Notation — lightweight data interchange format used for API payloads and storing structured data in MySQL JSON columns. |
| **JWT** | JSON Web Token — a compact, URL-safe token format used for stateless authentication. |
| **JTI** | JWT ID — a unique identifier embedded in each JWT, used for token revocation via Redis blocklist. |
| **Kafka** | A distributed streaming platform (mentioned as a skill in example job descriptions, not used in this project's infrastructure). |
| **LLD** | Low-Level Design — detailed design including class structures, API specs, sequence diagrams, and algorithms. |
| **LLM** | Large Language Model — an AI model trained on vast text data, capable of understanding and generating human language (e.g., GPT-4o). |
| **Location Score** | Component of the match score (15% weight) based on geographic alignment between job location and user location/remote preference. |
| **Match Score** | A weighted composite score (0–100) measuring how well a job posting aligns with a user's resume and preferences. |
| **MoSCoW** | A prioritisation technique: Must Have, Should Have, Could Have, Won't Have (this time). |
| **Microservices** | An architectural style where a system is composed of small, independently deployable services, each responsible for a specific domain. |
| **Nginx** | A high-performance reverse proxy and web server used for TLS termination and routing in this project. |
| **OpenPDF** | An open-source Java library (fork of iText 4) for generating PDF documents. Used to render tailored resumes and cover letters. |
| **ORM** | Object-Relational Mapping — technology that maps Java objects to database tables (Hibernate). |
| **PDF** | Portable Document Format — the file format used for generated resumes and cover letters. |
| **Persona** | A fictional representation of a target user group, used to guide product decisions. |
| **PII** | Personally Identifiable Information — data that can identify an individual (name, email, phone). Governed by GDPR. |
| **Playwright** | A browser automation library used to scrape JavaScript-rendered job listing pages. |
| **PRD** | Product Requirements Document — defines user stories, MoSCoW priorities, and UI wireframes. |
| **RabbitMQ** | A message broker that implements AMQP. Used to decouple pipeline stages with queues and exchanges. |
| **Refresh Token** | A longer-lived JWT (7 days) used to obtain new access tokens without re-authentication. |
| **Resume Tailoring** | The AI-driven process of modifying a user's resume to incorporate JD-specific keywords and emphasise relevant experience. |
| **REST** | Representational State Transfer — an architectural style for APIs using HTTP methods (GET, POST, PUT, DELETE). |
| **Redis** | An in-memory data structure store used for JWT blocklisting, rate limiting, and caching. |
| **SRS** | Software Requirements Specification — details functional and non-functional requirements with priority levels. |
| **Skill Score** | The highest-weight component of the match score (50%) based on the overlap between resume skills and required job skills. |
| **Spring Boot** | A Java framework for building production-ready web applications with minimal configuration. |
| **Spring Security** | A Spring module providing authentication, authorisation, and security features. |
| **Springdoc OpenAPI** | A library that auto-generates OpenAPI 3 (Swagger) documentation from Spring Boot annotations. |
| **Testcontainers** | A Java library that spins up Docker containers during tests, providing real databases and services for integration testing. |
| **TLS** | Transport Layer Security — cryptographic protocol ensuring secure communication over a network. Version 1.3 required. |
| **Token Blocklist** | A Redis set storing revoked JWT JTIs to prevent reuse of logged-out tokens. |
| **utf8mb4** | MySQL character set that supports the full Unicode range including emoji, required for job descriptions from global sources. |
| **Wiremock** | A tool for mocking HTTP services (e.g., OpenAI API) during testing. |
